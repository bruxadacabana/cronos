1. INSTALAÇÃO DE PRÉ-REQUISITOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌─ WINDOWS 10 / 11 ──────────────────────────────────────────────────────────┐

  1. Instale o Python 3.10 ou superior:
     https://www.python.org/downloads/windows/
     ⚠  Durante a instalação, marque "Add Python to PATH"

  2. Verifique a instalação abrindo o Prompt de Comando (cmd):
       python --version
       pip --version

  3. (Opcional) Instale o Ollama para análise com IA:
     https://ollama.ai/download/windows
     Após instalar, abra o cmd e execute:
       ollama pull llama3

  4. (Opcional) Instale as fontes do Cronos:
     Abra o PowerShell como Administrador e execute:
       Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
       cd C:\caminho\para\cronos
       install_fonts.ps1
     OU instale manualmente os .ttf da pasta src/assets/fonts/

└────────────────────────────────────────────────────────────────────────────┘

┌─ LINUX — CachyOS / Arch Linux ─────────────────────────────────────────────┐

  1. Instale as dependências do sistema:
       sudo pacman -S python python-pip python-virtualenv
       sudo pacman -S qt6-base qt6-tools
       sudo pacman -S noto-fonts ttf-liberation  # fontes (opcional)

  2. (Opcional) Instale o Ollama:
       curl -fsSL https://ollama.ai/install.sh | sh
       ollama pull llama3

  3. Não é necessário mais nada — o cronos.sh cuida do resto.

└────────────────────────────────────────────────────────────────────────────┘

┌─ LINUX — Ubuntu / Debian / Linux Mint ─────────────────────────────────────┐

  1. Instale as dependências do sistema:
       sudo apt update
       sudo apt install python3 python3-pip python3-venv
       sudo apt install libxcb-cursor0 libxcb-xinerama0  # necessário para PyQt6

  2. (Opcional) Instale o Ollama:
       curl -fsSL https://ollama.ai/install.sh | sh
       ollama pull llama3

  3. Não é necessário mais nada — o cronos.sh cuida do resto.

└────────────────────────────────────────────────────────────────────────────┘

┌─ ANDROID (Termux — experimental) ──────────────────────────────────────────┐

  1. Instale o Termux pela F-Droid (não pela Play Store):
     https://f-droid.org/packages/com.termux/

  2. No Termux:
       pkg update && pkg upgrade
       pkg install python python-pip x11-repo
       pkg install termux-x11-nightly
       pip install PyQt6 --extra-index-url https://pypi.org/simple/

  3. Para exibir a interface gráfica:
       Instale o Termux:X11
       Execute: export DISPLAY=:0
       Inicie o Termux:X11 e então: python cronos.py

  ⚠  A experiência no Android é experimental. Resolução e toque podem
     não funcionar perfeitamente. Recomendado apenas para uso avançado.

2. INICIALIZAÇÃO DO PROGRAMA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌─ WINDOWS ───────────────────────────────────────────────────────────────────┐

  Método 1 — Duplo clique (mais fácil):
    Duplo clique em  cronos.bat

  Método 2 — Prompt de Comando:
    cd C:\caminho\para\cronos
    cronos.bat

  Método 3 — PowerShell:
    cd C:\caminho\para\cronos
    python cronos.py

  Modo debug (gera logs mais detalhados):
    python cronos.py --debug

  ℹ  Na primeira execução, o cronos.bat cria automaticamente o ambiente
     virtual (.venv) e instala todas as dependências. Isso leva alguns
     minutos. As execuções seguintes iniciam em segundos.

└────────────────────────────────────────────────────────────────────────────┘

┌─ LINUX / CachyOS ───────────────────────────────────────────────────────────┐

  Método 1 — Terminal (mais fácil):
    cd /caminho/para/cronos
    bash cronos.sh

  Método 2 — Duplo clique no gerenciador de arquivos:
    Clique direito em cronos.sh → "Executar como programa"
    (dependendo do DE, pode ser necessário habilitar execução)

  Método 3 — Manual:
    cd /caminho/para/cronos
    python3 -m venv .venv
    source .venv/bin/activate
    pip install -r requirements.txt
    python3 cronos.py

  Modo debug (gera logs mais detalhados):
    python3 cronos.py --debug

┌─ WINDOWS ───────────────────────────────────────────────────────────────────┐

  Opção A — Menu Iniciar:
    1. Clique direito em cronos.bat → "Criar atalho"
    2. Mova o atalho criado para:
         %APPDATA%\Microsoft\Windows\Start Menu\Programs\
       (Cole esse caminho na barra do Explorador de Arquivos)
    3. O Cronos aparecerá no Menu Iniciar

  Opção B — Área de trabalho:
    Clique direito em cronos.bat → Enviar para → Área de Trabalho (criar atalho)

  Opção C — Barra de tarefas:
    Clique direito no atalho → Fixar na barra de tarefas

┌─ LINUX (GNOME, KDE, XFCE, LXDE — qualquer DE com suporte .desktop) ────────┐

  1. Edite o arquivo cronos.desktop na pasta do programa:
     Abra-o num editor de texto e substitua:
       /CAMINHO/PARA/CRONOS
     pelo caminho real onde você extraiu o programa, por exemplo:
       /home/usuario/cronos

  2. Copie para a pasta de aplicativos do usuário:
       cp cronos.desktop ~/.local/share/applications/

  3. Atualize o banco de dados de aplicativos:
       update-desktop-database ~/.local/share/applications/

  4. O Cronos aparecerá no launcher/menu de aplicativos do seu DE.

  ℹ  Para instalar para todos os usuários do sistema (requer root):
       sudo cp cronos.desktop /usr/share/applications/

4. FONTES DE NOTÍCIAS (data/sources.json)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  A lista de fontes RSS é armazenada em:
    data/sources.json

  Esse arquivo é criado automaticamente na primeira execução com 25 fontes
  padrão. Você pode editá-lo manualmente com qualquer editor de texto.

  Formato de cada fonte:
    {
      "name":           "Nome da fonte",
      "url":            "https://exemplo.com/rss",
      "category":       "brasil|internacional|tecnologia|ciencia|economia|...",
      "language":       "pt|en|es",
      "country":        "BR|US|GB|...",
      "economic_axis":  -1.0 a 1.0,   // negativo=esquerda, positivo=direita
      "authority_axis": -1.0 a 1.0    // negativo=libertário, positivo=autoritário
    }

  ⚠  Substack foi removido da aba Fontes RSS (v1.5). Newsletters do Substack
     devem ser configuradas na aba Redes Sociais.

  ⚠  Alterações no sources.json só são sincronizadas com o banco de dados
     na próxima inicialização. Para adicionar fontes em tempo real, use
     a aba Fontes dentro do programa.

5. FUNCIONALIDADES v1.5 (NOVIDADES)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  BARRA DE PROGRESSO DE ANÁLISE MELHORADA:
    - Exibe a contagem real de artigos em tempo real:
        "Analisando 12 de 45 artigos  (33 restantes)"
    - Barra de progresso maior (260-380px × 18px) com porcentagem decimal:
        "67.8%"
    - Progresso combina pré-análise + análise completa numa única barra
    - Mini barra de progresso no leitor, ao lado de "⏳ Analisando...",
      enquanto a IA processa o artigo aberto

  SISTEMA DE LOGS DE ANÁLISE (data/logs/):
    - data/logs/cronos.log      — log geral da aplicação (INFO+)
    - data/logs/analysis.log    — log detalhado das análises de IA (DEBUG+)
    - Rotação automática: máximo 2 MB por arquivo, 5 backups mantidos
    - Cada análise registra: título, tempo em segundos, categoria,
      tom emocional, clickbait score e prévia do resumo
    - Erros têm separador visual e traceback completo
    - Cada chamada ao Ollama registra: tamanho do prompt, max_tokens,
      timeout configurado e tempo real de resposta
    - JSON inválidos da IA registram os primeiros 200 chars da resposta bruta
    - Modo debug: python cronos.py --debug
        ativo: logs de cada chunk individual e chamadas Ollama no cronos.log

  TRADUÇÃO SEM OLLAMA — 4 BACKENDS GRATUITOS EM CASCATA:
    1. deep-translator (Google Translate via scraping)
       → instale com: pip install deep-translator
       → mais preciso e rápido; requer conexão com internet
    2. MyMemory API
       → sem instalação; limite ~1000 palavras/dia por IP
    3. Lingva (instâncias públicas open-source)
       → sem instalação; sem limite declarado; múltiplas instâncias de fallback
    4. ArgosTranslate (100% offline)
       → instale com: pip install argostranslate
       → requer download do pacote de idioma correspondente
       → funciona sem internet após instalação
    → Se todos falharem, mensagem clara com instruções de instalação

  ANÁLISE DE IA MAIS RÁPIDA:
    - Conteúdo limitado a 5.000 chars antes de chunkar (suficiente para artigos)
    - chunk_size reduzido de 15.000 → 4.500 chars (adequado para llama3:8b)
    - max_tokens reduzido de 600 → 400 (o JSON de resposta cabe em ~300 tokens)
    - timeout reduzido de 90s → 60s (mais agressivo, evita travamentos)
    - num_predict agora é passado corretamente ao Ollama (era ignorado antes)
    - Verificação de disponibilidade do Ollama não é mais duplicada a cada chunk

  CAROUSEL / TRENDING ESTABILIZADO:
    - Correção de crash "RuntimeError: C/C++ object deleted" ao atualizar
      a aba de destaques quando o carrossel estava exibindo o placeholder
    - O placeholder é recriado a cada necessidade, nunca reutilizado após deleção

  CONTEÚDO "None" NO LEITOR CORRIGIDO:
    - Banco pode retornar None mesmo com fallback declarado; agora tratado
      explicitamente em load_article, _normalize_content e _build_html

8. INTELIGÊNCIA ARTIFICIAL (OLLAMA)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Instale: https://ollama.ai

  Modelos recomendados (execute no terminal após instalar o Ollama):
    ollama pull llama3        — rápido, ótimo para PT/EN (recomendado)
    ollama pull mistral       — ótimo para análise política
    ollama pull gemma2        — equilibrado
    ollama pull phi3          — leve, para máquinas com pouca RAM

  Configure em: Configurações → Inteligência Artificial
  Indicador na sidebar: ● verde = Ollama online, ● vermelho = offline

  DOIS NÍVEIS DE ANÁLISE:

    Pré-análise (rápida, ~3-5s por artigo):
      → Roda em batch para todos os artigos novos automaticamente
      → Usa apenas o título como entrada (não precisa de conteúdo completo)
      → Extrai: keywords, categoria, tom emocional, score de clickbait

    Análise completa (profunda, ~15-30s por artigo):
      → Disparada imediatamente ao abrir uma matéria (prioridade máxima)
      → Roda em background para artigos já pré-analisados (sem notícia aberta)
      → Preenche: resumo, implicações, 5Ws, eixo econômico, eixo autoritário
      → Conteúdo limitado a 5.000 chars para otimizar velocidade

  O Ollama é usado APENAS para análise de artigos.
  A tradução de textos usa backends separados (ver seção 9).

9. TRADUÇÃO DE ARTIGOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  A tradução funciona sem nenhuma configuração adicional, usando uma cadeia
  de 4 backends gratuitos em ordem de prioridade:

  BACKEND 1 — deep-translator (Google Translate)
    Requer:  pip install deep-translator
    Limite:  generoso (sem limite fixo conhecido)
    Requer internet: sim
    Recomendado para uso regular.

  BACKEND 2 — MyMemory API
    Requer:  nada (HTTP direto)
    Limite:  ~1.000 palavras/dia por IP (gratuito sem chave)
    Requer internet: sim
    Usado automaticamente se o backend 1 não estiver instalado.

  BACKEND 3 — Lingva (instâncias públicas)
    Requer:  nada (HTTP direto)
    Limite:  sem limite declarado
    Requer internet: sim
    Múltiplas instâncias de fallback para maior disponibilidade.

  BACKEND 4 — ArgosTranslate (offline)
    Requer:  pip install argostranslate
             + download do pacote de idioma desejado
    Limite:  nenhum (100% local)
    Requer internet: apenas para baixar o pacote de idioma pela 1ª vez
    Ideal para uso sem internet.

  Idiomas suportados: Português, Inglês, Espanhol, Francês, Alemão,
  Italiano, Japonês, Chinês, Árabe, Russo, Coreano, Holandês, Polonês,
  Turco, Ucraniano.

  Resultados de tradução são salvos em cache (data/cronos.db, tabela
  translations) para evitar chamadas repetidas ao mesmo artigo.

  ARQUIVOS DE LOG:

    data/logs/cronos.log
      Log geral da aplicação. Contém INFO de todos os módulos Cronos:
      feeds baixados, artigos salvos, ações do usuário, erros gerais.
      Rotação automática: 2 MB / 5 backups mantidos.

    data/logs/analysis.log
      Log dedicado às análises de IA. Contém DEBUG+ dos módulos:
      cronos.analyzer, cronos.ai, cronos.translator.
      Cada análise registra:
        - Início: ID do artigo e título
        - Por chunk: número do chunk, tempo em segundos
        - JSON inválido: primeiros 200 chars da resposta bruta da IA
        - Sucesso: categoria, tom, clickbait, prévia do resumo, tempo total
        - Falha: tipo de erro e traceback completo
      Cada chamada ao Ollama registra:
        - Tamanho do prompt (chars), max_tokens, timeout configurado
        - Tempo real de resposta em segundos
        - Motivo exato em caso de falha (timeout, rede, etc.)
      Rotação automática: 2 MB / 5 backups mantidos.

    error_log.txt  (na raiz do programa)
      Crashes fatais não capturados. Mantido por compatibilidade.

  MODO DEBUG:
    Execute com:  python cronos.py --debug
    Efeito: ativa nível DEBUG no cronos.log (logs de cada chunk individual
    e chamadas Ollama). Sem a flag, apenas INFO+ vai para o cronos.log,
    mas o analysis.log sempre recebe DEBUG+.

  EXEMPLO DE ENTRADA NO analysis.log:
    ────────────────────────────────────────────────────────────
    2025-01-15 14:23:11 [ERROR   ] cronos.analyzer
    [BATCH] Falha na análise (chunk 1)
      ID    : 1234
      Título: Exemplo de título do artigo
      Erro  : TimeoutError: Ollama não respondeu em 60s
    ────────────────────────────────────────────────────────────

11. PRIVACIDADE E DADOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Todos os dados ficam LOCALMENTE na pasta data/:
    data/cronos.db       — banco de dados (artigos, análises, arquivo pessoal,
                           cache de traduções)
    data/sources.json    — lista de fontes RSS (editável)
    data/settings.json   — preferências do usuário
    data/logs/           — logs de execução (cronos.log, analysis.log)
    data/cache/          — cache temporário de scraping

  Conexões externas realizadas pelo programa:
    - Requisições RSS para as URLs das fontes configuradas
    - Scraping de artigos completos (URLs das notícias)
    - API do Ollama (local, rodando na sua máquina — sem internet)
    - Tradução: MyMemory API ou Lingva (se deep-translator não instalado)
    - Tradução: Google Translate via deep-translator (se instalado)
    - API Feedly (opcional, busca de novas fontes RSS — somente leitura)
    - Redes sociais: apenas se configuradas na aba Redes Sociais

  O Ollama roda 100% localmente. Nenhum texto de artigo é enviado
  para servidores externos durante a análise de IA.

13. SOLUÇÃO DE PROBLEMAS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [Windows] Tela preta ou erro ao iniciar:
    → Certifique-se de que Python está no PATH
    → Abra o cmd e execute: python --version
    → Se não funcionar, reinstale o Python marcando "Add to PATH"

  [Windows] Fontes não aparecem corretamente:
    → Instale as fontes manualmente: clique duplo nos .ttf em src/assets/fonts/
    → Ou execute install_fonts.bat como Administrador

  [Linux] Erro "xcb" ou tela não aparece:
    → Ubuntu/Debian: sudo apt install libxcb-cursor0 libxcb-xinerama0
    → Arch/CachyOS: sudo pacman -S xcb-util-cursor

  [Qualquer OS] Fontes não carregam:
    → bash install_fonts.sh  (Linux)
    → Verifique se curl está instalado: curl --version

  [Qualquer OS] Ollama não conecta:
    → Verifique se está rodando: curl http://localhost:11434/api/tags
    → Inicie manualmente: ollama serve
    → Configure outra URL em Configurações → IA

  [Qualquer OS] Tradução não funciona:
    → Execute: pip install deep-translator
    → Se sem internet: pip install argostranslate
      (e baixe o pacote do idioma desejado)
    → Verifique data/logs/analysis.log para ver qual backend foi tentado

  [Qualquer OS] Análise de IA muito lenta:
    → Use llama3:8b (mais rápido): ollama pull llama3:8b
    → Verifique data/logs/analysis.log para ver o tempo de cada chunk
    → Execute com --debug para log mais detalhado:
        python cronos.py --debug

  [Qualquer OS] Banco corrompido:
    → Apague data/cronos.db (perde histórico e arquivo pessoal)
    → Na próxima execução é recriado automaticamente
    → ⚠  data/sources.json é preservado separadamente

  ONDE ENCONTRAR LOGS:
    → data/logs/cronos.log      — log geral
    → data/logs/analysis.log    — erros e tempos de análise IA
    → error_log.txt             — crashes fatais (na raiz do programa)
